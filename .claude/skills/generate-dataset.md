# Generate IBS Dataset via Claude Code

Generate medical training data with automatic LLM-based quality evaluation.

## Input

$ARGUMENTS should specify:
- **Limit** (optional): Total number of entries to generate (default: 10 per batch)

## Prerequisites

1. Check if `datasets/chunks/*.jsonl` exists (run chunking first)
2. If not, tell user to run:
   ```
   uv run python scripts/collect/fetch_sources.py --source all
   uv run python scripts/chunk/chunk_sources.py --min-words 100
   ```

## CRITICAL: Medical Reference

Before launching ANY sub-agent, read these files — they MUST be injected into every prompt:
- `reference/ibs_reference.md` — verified medical facts
- `reference/hallucination_blocklist.md` — known false claims to reject
- `reference/medication_whitelist.md` — only these medications are valid

The `build_prompt()` function in `scripts/generate/generate_batch.py` automatically injects all reference material into every prompt. Use it to generate prompts:

```
uv run python scripts/generate/generate_batch.py --start 0 --count 10
```

## Process

### Phase 1: Generate (batches of 10)

1. **Get prompts**: Run `generate_batch.py --start {S} --count 10`
   - Each prompt includes: source chunk + IBS reference + hallucination blocklist + medication whitelist
   - Each prompt specifies a conversation type (factual_qa, anxious_patient, myth_busting, safety_refusal, doctor_followup, multi_turn)
2. **Launch 5 sub-agents** (2 chunks each) with the generated prompts
   - Use `haiku` model for sub-agents
   - Each agent writes results to its own temp file (avoid concurrent write data loss)
   - Sub-agent instruction: "Generate the requested IBS training pair. Output ONLY the JSON object, nothing else."
3. **Collect results**: Validate JSON with `generate_batch.py --validate /tmp/results_{N}.jsonl`
4. Write valid pairs to `datasets/generated/batch_{N}.jsonl`

### Phase 2: Auto-Eval (Medical Filters + LLM Judge)

After each batch of 10 entries:

1. **Run automated medical filters first**:
   ```
   uv run python scripts/evaluate/medical_judge.py --input datasets/generated/batch_{N}.jsonl --filters-only
   ```
   - Medication whitelist check
   - Dosage rejection
   - Diagnostic claim detection
   - Red flag handling verification
   - Hallucination blocklist check

2. **Generate judge prompts** for entries that pass filters:
   ```
   uv run python scripts/evaluate/medical_judge.py --input datasets/generated/batch_{N}.jsonl
   ```
   - Batches 3 entries per judge prompt
   - Judge scores on: accuracy, safety, empathy, scope, completeness (1-5 each)

3. **Launch judge sub-agents** (use `haiku` model)
   - Each judges 3 entries
   - Each agent writes scores to its own temp file

4. **Score results**:
   ```
   uv run python scripts/evaluate/medical_judge.py --input datasets/generated/batch_{N}.jsonl --score /tmp/judge_results/
   ```
   - **Rejection thresholds**: safety < 4, accuracy < 3, or average < 3.5
   - Report: "X/10 passed filters, Y passed judge"

### Phase 3: First-Batch Checkpoint

After the FIRST batch of 10:
1. Show user 3 random accepted entries with their scores
2. Show pass rate and score distribution
3. **PAUSE** — ask user to confirm before continuing
4. If pass rate < 50%, stop and investigate prompt issues

### Phase 4: Scale Up

After user confirms first batch quality:
1. Continue generating in batches of 10
2. Track running totals: accepted / rejected / total
3. **Alert if any batch has >50% rejection rate** — stop and show examples
4. After reaching target count, run final dedup:
   ```
   uv run python scripts/evaluate/quality_filter.py --input datasets/validated/
   ```

## Sub-Agent Prompt Format

The prompts are generated by `generate_batch.py` which includes:
- Full IBS medical reference (verified facts)
- Hallucination blocklist (myths and false claims)
- Medication whitelist (only valid drugs)
- Conversation type instructions (factual, anxious patient, safety refusal, etc.)
- Hard rules: educational only, never diagnose, never prescribe dosages

Do NOT manually write sub-agent prompts. Always use `generate_batch.py` to ensure reference material is included.

## Output

Each batch writes to: `datasets/generated/batch_{N}.jsonl`
Validated entries go to: `datasets/validated/validated.jsonl`

After all batches complete and pass quality gate:
1. Merge all validated entries
2. Run `quality_filter.py` for final dedup
3. Spot-check 20 random entries from the merged file

## Quality Rules (NON-NEGOTIABLE)

- Every entry must pass automated medical filters AND LLM judge
- Safety score < 4 → REJECT (non-negotiable, medical safety)
- Accuracy score < 3 → REJECT
- Average of all 5 dimensions < 3.5 → REJECT
- First batch must be manually approved before scaling
- If >50% rejection rate in any batch, stop immediately
- Spot-check 20+ entries after generation
- If >5% of spot-checked entries contain medical errors, reject the entire batch

## Important Notes

- Use `haiku` model for ALL sub-agents (generation and judging)
- Each agent writes to its OWN file (prevents data loss from concurrent writes)
- Launch sub-agents in parallel batches of 5 for efficiency
- Track progress: show batch N/total, accepted/rejected counts
- The model must NEVER diagnose, prescribe specific dosages, or recommend stopping medications
- Red flag symptoms MUST always trigger "see a doctor" responses
